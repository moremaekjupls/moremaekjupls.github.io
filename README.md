# Khondamir Begmatov — личный сайт / personal site

Тёмный «космический» бренд-сайт на **Astro**: главная, кейс проекта **NURA**,
двуязычный блог (RU/EN) и страница «Обо мне». Дизайн по гайду `DESIGN.md`
(чёрный фон, акцент-violet, particle-констелляция, без теней и градиентов).

---

## 1. Как добавить пост в блог (без кода)

Блог собирается из markdown-файлов в папке **`src/content/blog/`**.
Один пост = **два файла**: русская и английская версии.

1. Скопируйте любой существующий пост, например `token-economics.ru.md`
   и `token-economics.en.md`.
2. Переименуйте оба под новую тему, например:
   `my-post.ru.md` и `my-post.en.md`.
3. В начале каждого файла (блок между `---`) заполните поля:

   ```yaml
   ---
   title: "Заголовок поста"
   excerpt: "Короткое описание на 1–2 строки для списка блога."
   pubDate: 2026-07-01          # дата YYYY-MM-DD
   lang: "ru"                    # "ru" в .ru.md, "en" в .en.md
   transId: "my-post"           # ОДИНАКОВЫЙ в обоих файлах = это и есть адрес /blog/my-post
   ---
   ```

4. Ниже пишите текст в markdown: `## Подзаголовок`, `**жирный**`,
   списки через `-`, цитаты через `>`, ссылки `[текст](https://...)`.
5. Сохраните, пересоберите/задеплойте (см. раздел 4). Пост появится автоматически:
   в списке блога, в «последних постах» на главной и по адресу `/blog/my-post`
   (RU) и `/en/blog/my-post` (EN).

> Чтобы временно скрыть черновик — добавьте в шапку `draft: true`.

## 2. Где менять ссылки и контакты

Все ссылки собраны в **одном** файле — `src/consts.ts`. Ищите слово `REPLACE`:

- `SOCIALS` — GitHub, X/Twitter, LinkedIn, e-mail (футер).
- `PROJECT.liveUrl` — рабочая ссылка на приложение NURA (кнопка «Открыть приложение»).
- `SITE.domain` — ваш домен (для SEO-тегов).

Тексты страниц (главная, About, кейс NURA, кнопки) лежат в `src/i18n/ui.ts` —
там всё на двух языках, RU и EN рядом. Меняете строку — меняется сайт.

Скриншоты NURA: замените плейсхолдеры в кейсе своими картинками
(положите файлы в `public/` и вставьте `<img src="/имя.png">` в
`src/components/views/ProjectView.astro`, блок «SCREENSHOTS»).

## 3. Структура

```
src/
  consts.ts              ← ссылки, соцсети, домен (REPLACE здесь)
  i18n/ui.ts             ← все тексты RU/EN
  content/blog/*.md      ← посты блога (по 2 файла на пост)
  content/config.ts      ← схема поста (не трогать без нужды)
  components/            ← Header, Footer, карточки, частицы
  components/views/      ← содержимое страниц
  layouts/Base.astro     ← общий каркас (head, шрифты, шапка, подвал)
  pages/                 ← маршруты: RU в корне, EN в /en
  styles/global.css      ← дизайн-токены и стили
public/                  ← favicon и статика
```

Язык по умолчанию — **русский** (адреса без префикса). Английский — под `/en`.
Переключатель RU/EN в шапке сохраняет текущую страницу.

## 4. Запуск и публикация

Нужен установленный **Node.js 18+**. В терминале, в папке проекта:

```bash
npm install      # один раз — поставить зависимости
npm run dev      # локальный просмотр на http://localhost:4321
npm run build    # собрать готовый сайт в папку dist/
```

**Деплой (рекомендуется Netlify или Vercel — бесплатно):**

1. Залейте папку проекта в репозиторий на GitHub.
2. На netlify.com / vercel.com → «New project» → выберите репозиторий.
3. Команда сборки: `npm run build`, папка публикации: `dist`.
4. Готово. Каждый новый пост в `src/content/blog/` после `git push`
   публикуется автоматически.

Сайт полностью статический — быстрый, без сервера и базы данных.

Палитра и типографика — из `DESIGN.md`. Шрифт Acronym недоступен публично,
поэтому подставлен близкий по характеру **Space Grotesk** (Google Fonts).

подключите через `@font-face`, имя семейства в CSS уже `--font-acronym`.

## 5. Редактирование блога в браузере (CMS)

Чтобы писать посты без файлов — подключён **Pages CMS** (бесплатно, без сервера):

1. Зайдите на **https://app.pagescms.org** → войдите через GitHub.
2. Выберите этот репозиторий. Конфиг `.pages.yml` уже в корне — CMS сам всё подхватит.
3. Появится визуальный редактор «Blog posts»: заголовок, дата, язык, текст.
   Сохранение = коммит в репозиторий → сайт пересобирается сам.

Важно про двуязычность: один пост = две записи (RU и EN) с **одинаковым `Post ID`**.
Создаёте русскую запись (язык `ru`), затем английскую (язык `en`) с тем же Post ID.

## 6. Контактная форма

Форма на странице «Обо мне» работает через **Formspree** (бесплатно, без бэкенда):

1. Зарегистрируйтесь на **https://formspree.io**, создайте форму, скопируйте её endpoint.
2. Вставьте его в `src/consts.ts` → `CONTACT.formspreeEndpoint`.
3. До этого форма скрыта автоматически. После — заявки приходят вам на почту.

## 7. Аналитика (необязательно)

Приватная аналитика без cookies. В `src/consts.ts` → `ANALYTICS`:
поставьте `enabled: true`, выберите `provider` (`plausible` или `goatcounter`)
и укажите `site`. Выключено по умолчанию.

## 8. Деплой на GitHub Pages

В репозитории уже лежит workflow `.github/workflows/deploy.yml` — он сам
собирает и публикует сайт при каждом `push` в ветку `main`.

Порядок:

1. Создайте репозиторий на GitHub. **Лучший вариант для чистых ссылок** — назвать его
   `<ваш-логин>.github.io` (тогда сайт открывается по адресу `https://<ваш-логин>.github.io/`).
2. Залейте код в репозиторий (ветка `main`).
3. В репозитории: **Settings → Pages → Build and deployment → Source: GitHub Actions**.
4. Через ~1 минуту сайт опубликуется. Дальше каждый пост из CMS публикуется автоматически.
5. В `astro.config.mjs` и `src/consts.ts` пропишите финальный адрес (`site` / `domain`).

> Если используете отдельный репозиторий БЕЗ своего домена (адрес вида `/имя-репо/`) —
> в `astro.config.mjs` поставьте `base: '/имя-репо'` и скажите мне: внутренние ссылки
> нужно чуть поправить под подпапку.

## 9. Посты с помощью ИИ (по запросу, бесплатно)

Без платного API. Когда есть новость или тема:
1. Откройте Cowork и пришлите ссылку/текст новости ассистенту.
2. Ассистент пишет пост на RU+EN в голосе блога и коммитит файлы
   `<slug>.ru.md` + `<slug>.en.md` в репозиторий.
3. Сайт пересобирается автоматически — пост выходит.
   (Если нужно сначала проверить — попросите положить как черновик `draft: true`,
   потом снимете флаг в CMS.)

Альтернатива без ассистента: написать пост вручную в Pages CMS (app.pagescms.org).
